# AgentForge — Claude Agent Team 構築・運用支援サービス 設計仕様

**作成日**: 2026-04-25
**作成者**: CEO Division (Agent Team)
**ステータス**: 承認済み・実装計画待ち

---

## 1. サービス概要

### コンセプト

Claude Code のエージェントチーム（CEO/Dev/Ops/Business/Leadership Division）の
設計・構築・運用を企業向けにサービスとして提供する。

無料OSSスキルの公開で認知・信頼を獲得し、有料コンサルティングへ転換する
フライホイールモデルを採用する。

### ミッション

「AIエージェントによる自律的な開発・運用を、内製化できない企業に届ける」

---

## 2. ターゲット顧客

### プライマリ

- 日本の中小IT企業・スタートアップ（エンジニア 3〜30名規模）
- AI自動化に興味はあるが、内製化リソースが不足している
- GitHub を使っている開発チーム

### セカンダリ

- 海外の個人開発者・小規模チーム（英語対応スキル経由で流入）

---

## 4. 提供価値（バリュープロポジション）

1. **即戦力**: 動いている実績（本リポジトリ）がそのままデモになる
2. **透明性**: OSSなので顧客がロックインを恐れない
3. **継続改善**: CEOエージェントが自律的に Issue を処理し続ける

---

## 5. Go-to-Market 戦略

### Phase 1: 認知獲得（Month 1〜2）

1. `claude-hurikaeri` を主要マーケットプレイスに登録
   - claudemarketplaces.com
   - skillsmp.com
2. README に英語サマリーを追加
3. Zenn / Qiita に技術記事を掲載（エージェントチームの仕組み・導入効果）
4. GitHub README に問い合わせ導線を設置（GitHub Discussions）

### Phase 2: リード獲得（Month 2〜3）

1. GitHub Discussions で問い合わせ受付開始
2. 初期 1〜3社を低額またはパイロット価格で受注し、ケーススタディ化
3. ケーススタディを Zenn/Qiita 記事として公開

### Phase 3: スケール（Month 4〜）

1. 保守契約顧客の積み上げ
2. 口コミ・紹介による新規獲得
3. 必要に応じてシンプルな LP（ランディングページ）を作成
